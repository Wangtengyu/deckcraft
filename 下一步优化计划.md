## 二、核心差距分析

### 1. 内容生成质量差距（严重）

**原skill做法**：
```
// 每页内容要详实：每个要点都要有具体说明
标题「微服务迁移五步法：小步快跑，持续验证」

步骤1：「评估现状」
全面梳理现有系统边界和依赖关系
识别核心业务模块与数据流向
评估团队技术栈和运维能力
```

**DeckCraft当前**：
```typescript
// 只是简单生成默认内容
function generateDefaultPoints(count) {
  const templates = [
    '核心要点一：具体描述说明',
    '核心要点二：数据支撑或案例',
  ]
}
```

**影响**：生成的PPT内容空洞，用户需要大量手动修改

---

### 2. 参考图功能未打通（严重）

**问题**：
- 前端可以上传图片，但没有上传到云存储的逻辑
- 后端代码已写好，但前端没有把图片URL传给后端

**需要做的**：
1. 实现图片上传到云存储
2. 获取图片URL后传给后端
3. 后端在Prompt末尾追加参考图说明

---

### 3. PPTX导出功能缺失（严重）

**原skill做法**：
- 使用 `pptxgenjs` 生成真实的PPTX文件
- 支持文字可编辑

**DeckCraft当前**：
- 只生成背景图片
- 没有生成PPTX的逻辑

**影响**：用户无法下载可编辑的PPT文件

---

### 4. 智能标题优化未实现（中等）

**原skill做法**：
```
❌ 纯描述：微服务架构迁移五步法
✓ 带观点：微服务迁移：小步快跑，持续验证

❌ 纯描述：第一步：评估现状
✓ 带观点：评估现状：摸清家底才能精准施策
```

**DeckCraft当前**：
- 前端有开关选项，但后端没有实现优化逻辑

---

## 三、下一步优化计划

### Phase 6: 完善内容生成（优先级：高）

**目标**：生成详实、有深度的PPT内容

**具体任务**：
1. 使用AI模型生成详细内容（调用Coze对话API）
2. 每个要点要有具体说明、数据支撑、案例
3. 标题要有观点、有吸引力

**预计工作量**：4-6小时

---

### Phase 7: 打通参考图功能（优先级：高）

**目标**：用户上传的图片能正确融入PPT

**具体任务**：
1. 实现图片上传到云存储（使用Laf的文件存储）
2. 获取图片URL后传给后端
3. 后端根据参考图模式生成正确的Prompt后缀

**预计工作量**：3-4小时

---

### Phase 8: 实现PPTX导出（优先级：高）

**目标**：用户能下载可编辑的PPT文件

**具体任务**：
1. 使用 `pptxgenjs` 库
2. 将生成的背景图片插入PPTX
3. 在PPTX中添加文字图层（可编辑）
4. 支持导出下载

**预计工作量**：6-8小时

---

### Phase 9: 实现智能标题优化（优先级：中）

**目标**：让标题更有观点和吸引力

**具体任务**：
1. 调用AI模型优化标题
2. 添加观点、数据、悬念等元素

**预计工作量**：2-3小时

---

### Phase 10: 完善模板复刻（优先级：中）

**目标**：支持上传PPTX文件直接分析

**具体任务**：
1. 实现PPTX转图片（使用pdf-lib或pptx2json）
2. 自动分析每页布局
3. 套用模板风格生成新PPT

**预计工作量**：8-10小时

---

## 五、快速见效路径

**如果时间有限，建议先完成 Phase 8（PPTX导出）**：

这是最关键的功能，没有它用户无法下载可用的PPT文件。

**实现步骤**：
1. 在 `generate-pptx` 函数中添加 pptxgenjs 逻辑
2. 将生成的背景图片URL插入PPTX
3. 添加文字图层（使用前端传来的内容）
4. 返回PPTX文件下载链接

## 六、测试发现的问题

1. **前端细分风格选择器未显示**
   - 可能是 app-v2.js 未正确加载
   - 需要检查浏览器控制台是否有错误

2. **参考图上传后无预览**
   - app-v2.js 中的 `handleRefImageUpload` 函数需要验证

3. **修改PPT和模板复刻入口不明显**
   - 前端需要添加更明显的入口

---

## 七、总结

当前 DeckCraft V6.0 已完成**基础功能框架**，但**核心生成体验**还需要优化：

1. **最紧急**：PPTX导出功能（用户无法下载可用文件）
2. **最重要**：内容生成质量（内容空洞影响实用性）
3. **有价值**：参考图功能（已写好代码，需要打通）

建议按上述顺序逐步完善。
