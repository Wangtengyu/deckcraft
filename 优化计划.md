# DeckCraft 优化计划

## 目标
复刻原 create-ppt skill 的核心功能，提升 DeckCraft 的专业性和实用性

---

## 当前差距分析

| 功能模块 | 原skill | DeckCraft | 差距程度 |
|---------|---------|-----------|---------|
| 五种主风格 | ✅ A/B/C/D/E | ✅ A/B/C/D/E | 无 |
| 细分风格 | ✅ 20+种细分风格 | ❌ 无 | **严重** |
| Prompt模板 | ✅ 详细模板+示例 | ⚠️ 简化版(80-120字) | **严重** |
| 参考图功能 | ✅ 4种参考模式 | ❌ 无 | **严重** |
| 修改PPT功能 | ✅ delete/modify/add | ❌ 无 | 中等 |
| 模板复刻 | ✅ 上传PPTX自动分析 | ❌ 无 | 中等 |

---

## Phase 1: 完善Prompt模板（优先级最高）

**原因**：影响最大、工作量最小、见效最快

### 1.1 优化现有Prompt模板

**当前问题**：
- 模板过于简化（80-120字），缺少关键约束
- 缺少禁用词列表（如"商务"、"现代"、"专业"等会导致人像生成）
- 缺少布局规范和示例

**优化方案**：
- 每种风格保留3个模板：封面页、内容页、结尾页
- 添加禁用词约束
- 添加布局说明（标题位置、内容区域、留白比例）
- 参考 `style_A_infographic.md` 等文件中的详细规范

### 1.2 添加详细示例

**参考原skill做法**：
- 为每种风格添加2-3个内容页示例（流程图、对比图、列表图等）
- 示例包含完整的prompt结构和内容描述

**预计工作量**：2-3小时
**影响范围**：所有PPT生成质量

---

## Phase 2: 添加细分风格配置（优先级高）

**原因**：细分风格能精准匹配用户场景，提升专业度

### 2.1 扩展 STYLE_CONFIG 结构

**原skill细分风格示例**：
- **A风格（信息图风）**：
  - 党政红金（党建/党课/主题党日）
  - 政务蓝（政务汇报/政策解读）
  - 文化古典（历史/国学/传统文化）
  - 米白暖色（互联网/科技/创意/教育培训）
  - 通用蓝白（传统行业/其他办公场景）

### 2.2 实现自动推荐逻辑

**方案**：
```typescript
const SCENE_KEYWORDS = {
  'A_party_red': ['党建', '党课', '主题党日', '表彰大会'],
  'A_gov_blue': ['政务', '政策', '政府', '机关'],
  'A_culture': ['历史', '国学', '传统文化', '古典'],
  'A_tech_warm': ['互联网', '科技', '创意', '教育培训'],
  'A_general_blue': [] // 默认
}
```

### 2.3 修改前端UI

- 风格选择后，显示推荐的细分风格
- 用户可选择覆盖推荐

**预计工作量**：4-6小时
**影响范围**：PPT视觉风格精准度

## Phase 3: 实现参考图功能（优先级中）

**原因**：用户上传图片后能更好地融入PPT

### 3.1 支持图片上传

**前端改动**：
- 在内容来源选项中添加"上传参考图"
- 支持多图上传（最多5张）

### 3.2 实现四种参考模式

| 模式 | 适用场景 | Prompt后缀 |
|------|---------|-----------|
| 配图嵌入 | 人物照、风景照、活动照 | `提供的参考图是XX，放置在XX位置。照片主体内容必须保持不变，可以添加与页面风格协调的相框或边框装饰。` |
| 严格原样 | Logo、产品图 | `提供的参考图是XX，放置在XX位置。必须作为独立图层原样嵌入，不添加任何边框或装饰，保持原始比例不拉伸。` |
| 内容参考 | 图表、架构图（重绘） | `提供的参考图是XX，作为内容参考，提取其中的XX信息结构，用当前页面的视觉风格重新绘制。` |
| 风格参考 | 设计稿（参考风格） | `提供的参考图是XX，作为风格参考，参考其配色/布局/风格应用到当前页面内容。` |

### 3.3 后端处理

- 图片上传到云存储，获取URL
- 将图片URL传入Coze工作流
- 在Prompt末尾追加参考图说明

**预计工作量**：6-8小时
**影响范围**：PPT内容丰富度

---

## Phase 4: 实现修改已有PPT功能（优先级中低）

**原因**：高级功能，但实现复杂度较高

### 4.1 前端改动

- 添加"修改已有PPT"入口
- 支持上传 .pptx.html 文件
- 显示PPT预览，允许用户选择要修改的页面
- 提供三种操作：删除、修改、新增

### 4.2 后端API设计

**新建API**：`/api/modify`

**请求参数**：
```json
{
  "source_file": "原文件.pptx.html",
  "operations": [
    { "action": "delete", "page_id": "2" },
    { "action": "modify", "page_id": "3", "suggestion": "修改建议" },
    { "action": "add", "insert_after": "1", "content": "新页面内容描述" }
  ]
}
```

### 4.3 实现逻辑

1. 解析 .pptx.html 文件，提取每页图片
2. 根据操作类型生成新的prompt
3. 调用Coze工作流重新生成受影响的页面
4. 合并生成新的 .pptx.html 文件

**预计工作量**：8-12小时
**影响范围**：PPT编辑能力

---

## Phase 5: 实现模板复刻功能（优先级低）

**原因**：高级功能，实现复杂度最高，需要AI模型支持

### 5.1 前端改动

- 在创建模式中添加"模板复刻"选项
- 支持上传 .pptx 或 .ppt 文件

### 5.2 后端处理

**方案A（推荐）**：使用AI模型分析PPT
- 将PPT转换为图片
- 调用多模态模型分析风格和布局
- 返回风格标签和每页布局信息

**方案B**：使用Python脚本（需要后端支持）
- 参考 `template_image_understanding.py` 脚本
- 分析PPTX文件结构

### 5.3 实现逻辑

1. 用户上传PPTX模板
2. 后端分析模板风格和布局
3. 自动匹配细分风格
4. 用户输入新内容，套用模板风格生成

**预计工作量**：12-16小时（不含AI模型接入）
**影响范围**：模板复用能力

## 执行建议

### 推荐顺序
1. **Phase 1** → **Phase 2** → **Phase 3** → **Phase 4** → **Phase 5**

### 快速见效路径
- 先完成 Phase 1 和 Phase 2，可显著提升生成质量
- Phase 3-5 可根据用户反馈逐步迭代

### 资源需求
- **Phase 1-2**：仅需前端+后端代码修改，无需额外资源
- **Phase 3**：需要云存储服务（Sealos已支持）
- **Phase 4-5**：需要文件解析和AI模型支持
